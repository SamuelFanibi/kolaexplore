# Figma Import Guide

## How to Use This Design Specification in Figma

### Step 1: Create Design System in Figma
1. Open Figma and create a new file
2. Create a "Design System" page
3. Set up your color styles using the color palette from the specification
4. Create text styles for all typography variants
5. Set up spacing and layout grids

### Step 2: Create Component Library
1. Create a "Components" page
2. Build reusable components:
   - Buttons (Primary, Secondary, Text)
   - Input Fields
   - Cards
   - Navigation Elements
   - Icons (if needed)

### Step 3: Create Page Designs
1. Create separate pages for each screen
2. Use the component library for consistency
3. Follow the layout structure guidelines
4. Implement responsive breakpoints using Figma's constraints

### Step 4: Set Up Auto Layout
- Use Figma's Auto Layout for flexible components
- Set up proper spacing and padding
- Enable responsive resizing

### Step 5: Create Prototypes
- Link frames together
- Add interactions and transitions
- Test user flows

## Figma Best Practices

### Naming Conventions
- Use forward slashes (/) for component organization: `Button/Primary`
- Use descriptive names: `Card/Product/Default`
- Use consistent spacing: `Spacing/8`, `Spacing/16`

### Component Structure
- Create base components first
- Use variants for different states
- Create component sets for variations

### Styles Organization
- Colors: `Color/Primary/500`
- Text: `Text/H1`, `Text/Body`
- Effects: `Shadow/Medium`

### Frame Organization
- Use clear page names
- Group related frames
- Use consistent frame sizes

## Export Settings
- Export assets at 1x, 2x, 3x for different screen densities
- Use SVG for icons
- Use PNG/JPG for images
- Set up export presets

## Collaboration
- Share with team members
- Use comments for feedback
- Version control with Figma's version history
- Use branches for major changes
